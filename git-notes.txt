Version Control With GIT

Plan -> Code -> Build -> Test -> Release -> Deploy -> Operate -> Monitor and the cycle repeates 


# Can be done offline

git init					- 	initialize empty git repository

Newfile (Working area) 		-	Untracked file
Add file to git repo		-	Add to git staging area
Commited to local git repo	- 	work tree is clean - no pending actions
------------------------------------------------------------------------ 
# Must be done online
Push local repo to remote repo



Commands

git version											-	Displays git version
git init 											- 	Initialize git repository
git status											- 	Displays the status of git repository
git add . / git add <filename>						-	Add all the in the repo when you specify (.) / add specific file in repo
git commit -m <message>								- 	Commit git repository by providing message		

git config --global user.email "you@example.com"	-	For the first time when you try to commit without providing email id & user name these cmds 												   are to be executed with proper email id and name and then commit.	
git config --global user.name "Your Name"			-

git log												-	Displays the history of the git commands executed
git config --list									-	Lists the configuration settings from config file in git repository
git config user.name								-	Displays the username from config file in git repository
git config user.email								-	Displays the mailid from config file in git repository
